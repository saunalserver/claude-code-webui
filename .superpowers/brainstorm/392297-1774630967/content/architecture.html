Architecture

System components and data flow

Core Components

Backend (server.js)
  • Express Server: HTTP server, static file serving, REST API
  • WebSocket Server: Real-time bidirectional communication
  • Session Manager: PTY process lifecycle management
  • Upload Handler: File upload, storage, serving

Frontend Components

Frontend (public/)
  • Terminal View: xterm.js terminal emulator
  • Tab Manager: Multi-session tab handling
  • Upload Sidebar: File upload UI with drag & drop
  • WebSocket Client: Real-time communication with backend

Data Flow

  1. User opens browser, connects via WebSocket
  2. Creates/attaches to terminal session
  3. Backend spawns PTY process with configured CLI
  4. Terminal output streamed back to frontend via WebSocket
  5. User input sent to PTY via WebSocket

Configuration

Environment variables (in .env):

  • CLI_COMMAND - CLI to run (e.g., claude, gemini, codex) default: claude
  • CLI_ARGS - CLI arguments (e.g., --dangerously-skip-permissions) default: --dangerously-skip-permissions
  • PORT - Server port default: 3420
  • UPLOAD_DIR - Directory for uploaded files default: ./uploads
  • MAX_FILE_SIZE - Max upload size in bytes default: 10485760 (10MB)
  • UPLOAD_TTL - Time to keep uploads in hours default: 48
