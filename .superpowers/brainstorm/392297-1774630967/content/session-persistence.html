Feature: Session Persistence

Reconnect to terminal without losing history

How it Works

  • Terminal output stored in memory (100KB buffer per session)
  • History persists across page refresh (up to 100KB)
  • On reconnect, frontend requests missed history from server
  • Server sends history to reconnected client
Session Lifecycle
  1. User creates/ connects via WebSocket
  2. Server creates PTY process with configured CLI
  3. Terminal streams to frontend via WebSocket
  4. Frontend stores history in memory (up to 100KB)
  5. On reconnect, server replays missing history and sends current history
  6. PTY exits, server kills PTY process,li>
